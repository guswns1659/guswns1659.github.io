## 암달의 법칙

- 프로파일링을 언제 하는가?
    - 성능(1건 당 응답 속도)이 느려졌을 때 원인을 파악하기 위해 한다.
    - 1건 당 오래 걸리는 메서드를 튜닝하는 것이 아니라 전체 시간이 오래 걸리는 메서드를 튜닝해야 한다.

- [암달의 법칙]([https://ko.wikipedia.org/wiki/%EC%95%94%EB%8B%AC%EC%9D%98_%EB%B2%95%EC%B9%99](https://ko.wikipedia.org/wiki/암달의_법칙))이란
    - 시스템의 일부를 개선했을 때 전체 시스템의 성능이 얼마나 향상되는지 계산하는 수식
    - 하지만 아무리 병렬화를 진행해도(코어를 늘려도) 시스템의 성능이 20배 이상 향상되지 못한다.

- 코어의 개수를 늘리더라도 성능이 개수만큼 늘어나지 않는 이유는 무엇일까?
  - 동기화 구간이 있기 때문에 성능이 안된다.

## 커리 최적화1

- 쿼리 튜닝은 안하는 게 좋다. 그럼 그 전에 성능을 개선할 수 있는 사항은?
    - 커넥션 수 조정
    - 하드웨어 수직 확장 (기존 인프라의 성능 향상)
    - DB 수평 확장은? (어렵다. 샤딩이나 Read 레플리카를 만들어야 한다.)

- 샤딩이란 무엇인가?
  - 같은 테이블의 데이터를 다수의 데이터베이스에 분산하여 저장하는 방법을 의미한다.

- 클러스터드 인덱스
    - MySQL의 저장구조이고 p.k에 대해서 인덱스를 생성하고 p.k의 인덱스(B-tree 구조)를 통해 데이터에 접근하는 방식.
- MySQL의 explain 명령은 DB의 실행계획을 알려준다.
    - id는 하나의 실행 단위를 말한다.

## 쿼리 최적화2
- load data local infile로 데이터 삽입이 잘 들어가는데 테이블 컬럼 타입을 어떻게 했지?
- explain은 mysql 5.7이후 select 이외에 명령이 가능하다.
- explain의 id는 select 당 1개가 부여된다.
- dependant subquery란 무엇인가?
    - 서브퀴리인데, 바깥 테이블과 연관된 서브쿼리이다.
- derived2 테이블이란?
    - select id 2번으로 만들어지는 테이블
- TYPE 컬럼의 어떤 데이터가 나오는가?
    - 실제 데이터를 읽는 방법을 말한다.
    - System. const, ref, range, index, all 등이 있다.

- 복합인덱스를 제대로 사용하려면 어떻게 하는가?
    - 두 개를 다 쓸 때 효과적이다. 하나만 사용할 땐 mysql이 풀스캔할 수 있다.
    - 복합인덱스 2번째 인덱스 단독으로 사용할 수 없다.
- key_length는 무엇을 뜻하는가?
    - 단위가 바이트이고 인덱스를 얼마나 사용하고 있는가 나타낸다.
