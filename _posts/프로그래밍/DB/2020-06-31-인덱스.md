## 개요
인덱스, DB에 대한 호눅스 수업을 정리한 글

### 인덱스 관련
- 디스크 기반 자료구조와 메모리 기반 자료구조의 차이는?

	- 디스크 기반 자료구조는 컴퓨터가 꺼져도 데이터가 존재한다. 반면 메모리 기반 자료구조는 컴퓨터가 꺼지면 데이터가 사라진다.

- BST는 어떤 자료구조이고 순회 방식은 몇 가지인가?

	- BST는 부모노드 기준으로 왼쪽 노드는 부모노드의 값보다 작은 값이 오른쪽 노드는 부모 노드의 값보다 큰 값이 들어가는 자료구조이다. 순회방식은 전위, 중위, 후위 3가지 방식이 있다.

- HashMaP과 BST의 차이는 무엇인가?

	- HashMap은 키와 값이 바로 연결되어 있어 평균적으로 O(1)이지만 BST은 키값에 맞는 값을 탐색하기 때문에 평균적으로 O(logn)이다.

- B-tree란 무엇인가?

	- 디비에서 검색을 편하게 해주는 자료구조이다. 파일에 읽고 쓸 수 있는 자료구조이다.
  - BST와 다르게 하나의 노드에 여러개의 데이터가 들어갈 수 있다.

![Btree자료구조](/assets/DB/Btree자료구조.png)

- B+tree
  - Btree와 다르게 루트 노드에서 데이트를 찾을 수 없지만 자식노드 간 이동이 가능하다.
![B+Tree](/assets/DB/B-Tree.png)


- Hash와 B-tree의 차이점은 무엇인가?
	- Hash는 조건 검색이 불가능하다.

- 인덱스란 무엇인가?

	- 데이터를 빠르게 검색할 수 있게 도와주는 자료구조이다.
  - 테이블이 만들어질 때 Primary key의 인덱스를 자동으로 만든다.

- 참조무결성 제약조건은 무엇인가?

	- 외래키는 Null이거나 참조 릴레이션의 기본키 값과 동일해야 한다.

### 추가) 무결성 제약 조건
- 개체 무결성
  - 기본키는 Null이거나 중복 값을 허용하지 않는다.
- 참조 무결성
  - 외래키는 Null이거나 참조 릴레이션의 기본키 값과 동일해야 한다.
- 도메인 무결성
  - 각 필드의 속성은 도메인의 범위 내에 있어야 한다. ex) 중학교 학년이란 컬럼의 값들은 1~3사이만 가능.
- 고유 무결성
  - 특정 컬럼이 고유한 값만 가지도록 되어 있다면 그 속성값은 모두 달라야 하는 제약조건을 의미한다.
- Null 무결성
  - 특정 컬럼에 NUll이 허용되지 않는다면 Null 값이 올 수 없는 조건을 의미한다.

- InnoDB의 자료구조는 어떤 자료구조이고 그에 대한 설명을 해보시오.

	- Clustered Index이다. Primary key에 대해서 B+ tree를 만들고 data가 순차적으로 연결되어 있다.

- RID란 무엇인가?

	- PK 인덱스에 저장된 value이다. key는 Primary Key이다. 하지만 page번호만으로는 특정 데이터 하나를 찾을 수 없기 때문에 page번호와 그 페이지 내 데이터의 위치, slot 번호가 적혀있다.

- 레코드를 저장하는 Page는 어떻게 생겼는가?

	- 16kb 크기 제한으로 레코드가 담겨져 있는데 마지막 칸에는 각 레코드가 시작되는 바이트 크기가 적힌 slot이 있다.. 단순히 RID 1,5로 찾기는 힘들다. 왜냐하면 각 레코드는 가변길이이기 때문이다.
  - 만약 레코드가 삭제되면 마지막 칸에도 위치 정보가 삭제된다. 시간이 지나면 가비지 컬렉터 역할하는 기능이 동작해 삭제된 레코드를 지운다.

- Secondary Index는 어떻게 동작하는가?
  - 바로 데이터에 접근하지 않고 Primary Index를 통해 한번 거쳐서 접근한다.
  - Key는 해당 인덱스이지만 value는 RID가 아니라 PK가 들어간다. 즉, PK 인덱스를 통해서 데이터에 접근한다.

- 모든 컬럼에 대한 인덱스를 만들면 안되나?
  - 인덱스가 많이 만들어지기 때문에 저장공간를 차지한다.

- 인덱스를 사용하기 좋은 조건은 무엇인가? (Selectivity)

  - 데이터의 크기는 방대한데 매 검색하는 조건이 작은 경우 (전체 데이터의 10%)

- OLTP (Online Transactional Processing)과 OLAP(Analytical)의 차이는?

  - 트랜잭션(CRUD) 관련 데이터를 처리하는 프로세싱
  - 데이터를 분석하는 프로세싱

- 커버링 인덱스란 무엇인가?
  -  SELECT, WHERE, ORDER BY, GROUP BY 등에 사용되는 모든 컬럼이 인덱스의 구성요소인 경우
  - 데이터로 접근하지 않고 인덱스로만 접근하기 때문에 쿼리 속도가 빠르다.

- 복합인덱스의 순서가 중요한 이유는?
	- 처음 인덱스가 2번째 인덱스를 거쳐서 Primary Index로 접근하기 때문에 만든 순서대로 이용해야지 인덱스를 사용한다.
