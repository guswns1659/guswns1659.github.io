## 스웨거

- swagger 설정 파일 추가
  - host에 host 주소를 추가한다. 그래서 응답 결과가 정상적으로 온다.

```
@Configuration
@EnableSwagger2
public class SwaggerConfig {

	@Bean
	public Docket api() {
		return new Docket(DocumentationType.SWAGGER_2)
			.host("localhost:8080")
			.select()
			.build();
	}
}
```


## 테스트 코드 작성
테스트 코드 작성을 통해 TDD를 실천한다. 컨트롤러부터 레포지토리테스트까지 전부한다.

### 컨트롤러 테스트

- WebTestClient 주입하기
  - 테스트 코드에 아래 두 설정을 추가한다.
```
@SpringBootTest(webEnvironment = SpringBootTest.WebEnvironment.RANDOM_PORT)
@AutoConfigureWebTestClient(timeout = "30000")

@Autowired
private WebTestClient webTestClient;
```

- WebTestClient 사용하기

```
Company company = webTestClient.get()
            .uri(url)
            .exchange()
            .expectStatus().isEqualTo(HttpStatus.OK)
            .expectBody(Company.class)
            .returnResult()
            .getResponseBody();
```

## 깃허브 액션 배포
깃허브 액션을 이용해 자동배포하기 위해선 ec2, s3, codedeploy를 사용한다. 이 때 각 서비스에 설정이 필요하다.

### ec2 설정

- jdk 설치

```
sudo apt install openjdk-8-jdk
java -version
```

- 배포를 도와주는 CodeDeploy가 동작하기 위해서 ec2에 설치한다.

```
sudo apt-get update
sudo apt-get install ruby
sudo apt-get install wget
cd /home/ubuntu
wget https://aws-codedeploy-ap-northeast-2.s3.ap-northeast-2.amazonaws.com/latest/install
chmod +x ./install
sudo ./install auto
```

- codedeploy-agent가 제대로 설치되면 아래 1번째 명령어로 확인한 뒤 아직 inactive면 2번째 명령어를 입력한다.
```
sudo service codedeploy-agent status
sudo service codedeploy-agent start
```
