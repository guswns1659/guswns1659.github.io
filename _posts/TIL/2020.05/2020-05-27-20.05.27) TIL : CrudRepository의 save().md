### CrudRepository의 save()가 안될 때 

CrudRepository save() error 

- ```java
    bad SQL grammar [UPDATE `airbnb` WHERE `airbnb`.`id` = ?]; nested exception is java.sql.SQLSyntaxErrorException: You have an error in your SQL syntax; check the manual that corresponds to your MySQL server version for the right syntax to use near 'WHERE `airbnb`.`id` = 1' at line 1
    ```

- Spring Boot 실행할 때 데이터를 넣지 않고 datagrip으로 데이터를 따로 넣어서 문제가 발생하나 싶다. -> X

- user, accommdation에 reservation을 저장한 뒤에 airbnb에서 데이터를 변경한 뒤 저장해야 하나? -> X 

- 다른 CrudRepository만들어서 테스트하니 된다.  

- 원인 : 객체의 필드가 id 1개만 있다면 save()가 정상적으로 동작하지 않는다. id외 다른 필드도 필요하다.

- 테이블 reservation과 reservation_date를 하나로 합치고 1 대 N으로 가도 좋을 것 같다. 

### 배포 후 문제 발생 시 로그 확인하는 법

- tail -f /build/libs/nohup.out 치면 스프링 에러를 알수 있다. 
- nginx /var/log/nginx에서 에러 발생할 때 확인하는 법 
    - tail -f error.log
- RDS에 생성된 테이블의 필드와 스프링에서 수정한 필드가 다를 때도 서버 배포 시 에러 발생 가능 
- sql의 logging level을 보려면 logging.level.sql=debug를 추가
