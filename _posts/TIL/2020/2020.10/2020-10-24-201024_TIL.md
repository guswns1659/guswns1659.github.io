## 컴포넌트 스캔
- 컴포넌트 스캔은 @Component가 붙은 클래스를 빈으로 등록한다.
- 해당 클래스에 의존관계를 어떻게 주입하냐면 @Autoweird로 주입한다. 기존에 AppConfig를 이용할 때는 내가 빈을 직접 등록하고 해당 클래스의 의존관계를 직접해줬다. 컴포넌트 스캔은 스프링이 대신 해주는 것

# 객체지향 생활체조 9가지 원칙

소트웍스 앤솔러지 책에서 제프 베이가 제안한 원칙입니다.

1. 메서드는 한단계 들여쓰기만 한다. -> 들여쓰기 여러개 있으면 여러가지 역할을 할 가능서이 있다.
2. else 예약어를 사용하지 않는다. -> 간결함
3. 모든 원시값과 문자열을 포장한다 -> 행위를 추가할 수 있따. ex) Money
4. 한줄에 한점만 찍는다. -> 캡슐화가 깨진 상태일 수도 있다.
5. 줄여쓰지 않는다 -> 명확하게 의미를 보여준다.
6. 모든 엔티티를 작게 유지한다. -> 클래스가 한가지 일만 할 수 있도록
7. 2개 이상의 인스턴스 변수를 가진 클래스를 쓰지 않는다.
8. 일급컬렉션을 쓴다.
9. getter/setter/property를 쓰지 않는다.

-> 적용한 코드를 한번 보여줘라 : 에어비앤비 AccommodationService
-> 자주 적용하는 원칙은? 1. else예약어, 2. 일급컬렉션 3. 한줄에 한점만 찍는다.

# 전략 패턴

- 전략 패턴 : 각 객체가 할 수 있는 행위를 명세한 인터페이스가 있고 전략 클래스는 명세를 구현합니다. 행위를 바꾸고 싶을 때 전략을 바꾸는 코드로 변경할 수 있습니다.

예시 : 자바의 Comparator의 compare(), httpServlet
