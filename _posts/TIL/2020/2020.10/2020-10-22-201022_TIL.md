## 프로세스와 스레드 차이 (운영체제)
> 왜 질문 : 운영체제에서 동작하는 프로그램의 기본 원리를 잘 알고 있다면 어느 프로그램을 작성하든 직간접적으로 효과적인 구조를 만들 수 있다고 생각하기 때문에 묻는 질문

- 프로세스는 운영체제로부터 자원을 할당받아 실행하고, 스레드는 프로세스로부터 자원을 할당받아 실행
- 하나의 프로세스 안에서 여러 스레드 생성이 가능하며
- 각 스레드는 개별 스택을 가지고, 프로세스는 전역 메모리 공간을 공유하며 프로그램을 실행

## 동기식과 비동기식의 차이점 / 장단점
> 실무에서 막대한 요청, 데이터를 처리하려면, 동기식/비동기식을 고려해야 하기 때문이다. 아래 질문은 클라이언트 관점이다. 서버 관점은 다른 관점으로 생각해야 한다.

- 동기식 처리는 요청에 대한 응답을 기다린 후, 응답이 오면 다음 요청을 하는 방식이며
- 비동기식 일처리는 요청에 대한 응답을 기다리지 않고 다음 동작을 진행한다는 점

- 동기식의 장단점은 구성이 단순하고, 순서대로 실행 가능, 하지만 여러 일을 동시에 수행하는 멀티태스킹은 불가
- 비동기식은 동시에 여러 일을 수행할 수 있지만, 데이터의 정합성이 맞지 않는 문제가 발생하면 심각한 버그가 발생할 수 있습니다.

## 데이터베이스에서 인덱스를 사용하는 이유와 장단점
> 데이터의 양이 이전과 달리 막대하므로, 인덱스는 필수이다.

- 인덱스는 데이터를 논리적으로 정렬해서 검색과 정렬 속도를 높이기 위해 사용
- 단, 데이터 삽입, 변경이 수시로 일어나면 매번 인덱스를 변경해야 하므로, 성능 저하를 막기 위한 고려가 필요함.
- 인덱스를 사용하기 좋은 조건은 데이터의 양은 방대한데 검색하는 조건이 적은 경우가 효과적입니다. (10%로 알고 있습니다.)

## Redis와 mongodb에 대해 간략히 설명하세요.
> 최신 기술에 대한 관심이 있는지를 판단하기 위한 질문

- 둘 다 NoSQL 방식을 사용합니다. Mongodb가 document 형식으로 데이터를 저장하고, Redis는 key-value 형식으로 데이터를 저장함
- Redis는 인메모리DB로 데이터를 메모리에 저장하고 관리하기 때문에 성능이 좋지만 데이터를 유한하게 저장하기 때문에 캐쉬 등에 이용된다.
- Mongodb는 영속적인 스토리지에 저장한다. NoSQL 방식으로 사용됩니다.

## 스크립트 언어와 컴파일 언어를 나열하고 차이점을 설명하세요.
> 왜 질문 : 프로그래밍 언어와 컴파일러에 대해 알고 있는지

- 스크립트 언어로는 파이썬, 루비 등이 있으며, 컴파일 언어로는  C, Java 등이 있습니다.
- 컴파일 언어는 컴파일러를 통해 사전에 컴파일되어, 기계어 상태로 실행되므로 실행이 빠름. 최적화 작업도 진행
- 스크립트 언어는 컴파일 단계 없이 실행 단계에서 한 줄씩 기계어로 번역 후 실행되니 실행이 느리다.


## TCP와 UDP의 차이점과 장단점

- TCP와 UDP는 모두 OSI 7계층 중 전송 계층에서 사용되는 프로토콜입니다. TCP와 UDP의 가장 큰 차이점은 커넥션입니다. TCP는 3-way 핸드쉐이크 방식으로 커넥션을 맺습니다. 이유는 패킷(데이터) 안정적으로 전송하기 위함입니다. UDP는 커넥션을 맺지 않고 패킷을 빠르게 전송해야할 때 사용됩니다. 장단점은 TCP는 데이터가 안정적으로 전송된다는 점. UDP는 빠르게 전송된다는 점. TCP는 메일, 웹브라우저 등에서 사용하고, UDP는 DNS조회, 영상 등에 사용됩니다.

## 3-way handshake는 무엇이고 어떤 방식으로 이루어지나요?
- TCP 프로토콜에서 서버와 클라이언트가 커넥션을 맺을 때 사용하는 방식입니다. 먼저 클라이언트가 서버에게 연결을 하겠다는 데이터를 보냅니다. 이 때 TCP 헤더의 SYN 컨트롤 비트는 1로 표시합니다. 이후 서버가 클라이언트의 패킷을 받았다는 표시로 ACK를 1로, 연결됐다는 표시로 SYN을 1로 표시한 뒤 송신합니다. 이후 클라이언트는 최종적으로 확인했다는 의미로 ACK를 1로 표시해 전달합니다.

# 우분투 마우스가 안 움직일 때
- 아래 명령어를 입력하고 다시 껐다 킨다.

```java
gsettings set org.gnome.settings-daemon.plugins.cursor active false
```

- 명령이 안되면 아래 명령어로 sudo를 업데이트 한다.

```java
sudo pat-get udpate && sudo apt-get upgrade
```
