
- s3 버킷의 권한을 퍼블릭으로 해야 한다. 아닐 시 access denied 403 에러 발생

- (MuitipartFile -> File)로 변환하는 중 에러 발생해서 내가 짠 첫번째 실습코드로 변경함. -> 이후에 다시 시도하니 `Unable to calculate MD5 : AWS S3 bucket`가 발생해서 기존 코드로 변경했지만 (multipartFile -> File)로 만들 때 " new"를 추가해서 이름을 붙임.
- 기존에는 2개 파일의 이름이 같으니까 file.createNewFile()이 false가 나와서 에러가 발생했다. 이름을 다르게 하니까 정상동작한다. 2번째 코드 참고
- [참고 : 스택오버플로우](https://stackoverflow.com/questions/24339990/how-to-convert-a-multipart-file-to-file)

```java
// 첫번째 코드
private Optional<File> convert(MultipartFile multipartFile) throws IOException {
        File convertFile = new File(multipartFile.getOriginalFilename());
        try {
            multipartFile.transferTo(convertFile);
            return Optional.of(convertFile);
        } catch (Exception e) {
            return Optional.empty();
        }

```

```java
// 2번째 코드

private Optional<File> convert(MultipartFile multipartFile) throws IOException {
        File convertFile = new File(multipartFile.getOriginalFilename() + " new");
        if(convertFile.createNewFile()) {
            try (FileOutputStream fos = new FileOutputStream(convertFile)) {
                fos.write(multipartFile.getBytes());
            }
            return Optional.of(convertFile);
        }
        return Optional.empty();
    }
```

## The field data exceeds its maximum permitted size of 1048576 bytes.

- 업로드 파일의 크기가 해당 범위를 넘을 때 발생하는 에러. 아래 코드를 application.yml에 추가한다.
- [참고 : 스택오버플로우](https://stackoverflow.com/questions/37540028/how-to-set-the-max-size-of-upload-file)


```java
servlet:
    multipart:
      max-file-size: 10MB
      max-request-size: 10MB
```

- cloud.aw.s3.stack.auto: false 설정 없으면 로컬에서도 에러 발생
