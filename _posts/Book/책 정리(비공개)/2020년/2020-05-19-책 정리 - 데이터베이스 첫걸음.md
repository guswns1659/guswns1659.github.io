## 목차

### 1. 데이터베이스란
- 데이터베이스에게 요구되는 기능 4가지는 무엇인가? 
    - 1.데이터의 검색과 갱신이다. 방대한 데이터 속 사용자가 검색하고 싶은 데이터를 제공하는 기능을 가지고 있으며, 그 데이터의 등록, 수정 및 삭제도 가능하다. 
    - 2.동시성 제어이다. 개인 주소록과 다르게 데이터베이스는 여러 사용자가 접근한다. 여러 사용자가 하나의 데이터를 동시에 수정할 때 안전하게 수정할 수 있는 방법이 필요하다. 보통 누군가 수정하고 있다면 다른 사용자는 수정할 수 없게 막아 놓는다. 
    - 3.장애 대응이다. 데이터베이스는 장애에 강하게 대응할 수 있어야 한다. 수많은 데이터가 사라지지 않게 장애에 대응하는 것도 데이터베이스의 기능이다. 
    - 4.보안이다. 데이터베이스 내부에는 기밀이 보장되어야 하는 데이터가 존재할 수 있기 때문에 보안을 강하게 유지해야 한다.  

- DBMS와 데이터베이스의 차이는 무엇인가요? 
    - 데이터베이스는 기능과 구조를 나타낸 추상적인 개념이고 DBMS는 이걸 구현한 구체적인 소프트웨어이다. 
    
- 소프트웨어의 3가지 레이어는 각각 무엇이고 기능을 설명하세요
    - 운영체제 : 시스템이 동작할 수 있는 일종의 토대를 마련하는 소프트웨어이다. 
    - 미들웨어 : 운영체제 위에서 애플리케이션이 동작할 수 있게 지원하는 소프트웨어이다. 
    - 애플리케이션 : 업무용 기능을 가진 소프트웨어이다. 

### 3. 데이터베이스에 얽힌 돈 이야기 
- EOSL이란 
    - End Of Service Life이다. 서비스의 지원이 얼마나 남았는 지 알려주는 지표이다. 
    
### 4. 데이터베이스와 아키텍쳐 구성
- Stand-alone 이란 
    - DB가 네트워크에 접속하지 않고 물리적으로 독립되어 운영되는 상태
- Web 3계층이란
    - 클라이언트 요청을 처리하는 계층이 3개이다. 서버 계층 - 어플리케이션 계층 - DB 계층. 
- 클러스터란
    - 복수의 컴포넌트(사물)로 한 가지 기능을 담당하는 의미
- 신뢰성(Reliability)과 가용성(Availability)의 차이
    - 신뢰성 : 하드웨어나 소프트웨어가 고장나는 빈도. 
    - 가용성 : 사용자가 하드웨어나 소프트웨어를 사용할 수 있는 정도.

### 5. DBMS를 조작할 때 필요한 기본 지식
- 관계형 데이터베이스의 계층은 어떻게 나눠지는가? 
    - 스키마 : 여러개의 테이블 및 오브젝트(인덱스, 저장 프로시저 등)을 가지고 있다. 폴더와 비슷한 기능
    - 데이터베이스 : 복수의 스키마를 가지고 있다. 
    - 인스턴스 : 복수의 데이터베이스를 가지고 있다. 메모리를 실제로 사용하는 단위. 프로세스라고도 한다. 
    - 다만, MySQL과 Oracle은 스키마와 데이터베이스가 하나로 합쳐져있다. 
    
### 6. SQL 문의 기본 
#### 6-1 Select
- 중복된 값을 제거 하고 싶을 때 사용하는 문법은? 
    - distinct. ex) select name from user where rank > 6 and money > 1000 and money < 2000 limit 20;
- 쿼리 결과 개수를 알고 싶을 때 사용하는 문법은? 
    - count(*) ex) select count(*) from user where rank > 7 order by money limit 20;
- 쿼리 결과를 한 행으로 모아서 보고 싶을 때 사용하는 문법은? 
    - group_concat(column) ex) select group_concat(name) from user where rank >= 9;
- 데이터를 그룹화시켜서 보고 싶을 때 사용하는 문법은? 
    - group by column ex) select rank, count(*) from user where money > 100000 group by rank;
    - 주의) group by한 column으로 select을 해야 한다. 그룹화하지 않은 컬럼으로 select 적을 시 아래 에러가 발생한다. 
    ```mysql
    Expression #1 of SELECT list is not in GROUP BY clause and contains nonaggregated column 'csvdb.user.name' which is not functionally dependent on columns in GROUP BY clause; this is incompatible with sql_mode=only_full_group_by
    ```
 - ORDER BY로 정렬 시 주의해야할 점은? 
    - 정렬의 기준이 되는 컬럼은 Unique한 값으로 정해야 한다. 같은 값이 있으면 무작위로 순서가 정해진다.  
 - 데이터를 조작하고 계산하는 도구는 무엇이고 어떤 종류가 있는가?
    - 함수이며 2가지 종류가 있다.
    - 1.복수 행에 대해 집계를 수행하는 함수 ex) count, sum, avg, max, min .. 
    - 2.단일 행의 값에 대해 조작이나 계산을 수행하는 함수 
 - group_concat() 시 중복된 값을 제거하려면 어떻게 해야하는가? 
    - DISTINCT 문법을 사용해야 한다. 
    - select group_concat(DISTINCT name) from user where rank > 9; 
 - group by 사용 후 조건을 주고 싶을 때 어떤 문법을 사용하는가? 
    - Having 절을 사용한다. 
    - ex) select rank, count(*) from user where money > 100000 group by rank having rank > 4;
 - select문의 순서를 간략하게 표현하면 어떻게 되는가?
    - select - from - where - group by - having - order by 
 
 #### 6-2 Update, Insert, Delete
 - 테이블의 열 정보를 알 수 있는 문법은? 
    - show create table user\G (\G는 ; 대신에 사용할 수 있으며 결과를 세로로 보기 쉽게 해준다.)
 - Insert 시 Default의 기능은 무엇인가? 
    - 해당 컬럼 자료형의 기본값으로 자동으로 들어간다. 
    - 문자열은 빈 문자, 숫자형은 0, ID는 auto_increment된 값
 - MySQL에서 같은 테이블을 생성하는 문법과 데이터를 삽입하는 방법은?
    - create table usercopy like user;
    - insert into usercopy select * from user;
    - 주의) select절로 insert할 경우 서로 컬럼의 타입이 맞아야 한다.
