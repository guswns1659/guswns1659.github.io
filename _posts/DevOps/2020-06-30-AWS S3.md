## AWS S3

### 스토리지에 대한 기본 설명
- 스토리지에 반드시 저장할 수 없는 데이터는 무엇인가?

    - 다시 만들 수 있는 데이터.

- 시간이 지남에 따라 쌓이는 데이터를 무엇이라 하는가?

    - 시계열 데이터, 타임시리즈

- 분산시스템이 가져야할 중요한 2가지 특성은?

    - 내구성과 가용성.

- 분산 서버는 왜 홀수개로 유지하는가?

    - 원래 서버가 죽었는지 확인하는 마스터 서버가 있다. 하지만 마스터 서버도 죽으면 다른 서버의 생존을 확인할 수 없다는 문제가 발생한다. 그래서 요즘 방식은 하나의 서버가 죽었는지 안 죽었는지는 다른 서버들이 투표를 한다. 핑을 보냈는데 과반수로 반응이 오면 살았다고 판단한다.
    - 5개 중 1개가 죽은 상황에 한 노드의 데이터 변경 요청이 와서 살아있는 4개는 1로 바뀌고 죽은 1개는 그대로 0을 유지한다. 데이터의 불일치가 발생하기 때문에 서버들은 투표를 하고 과반수가 넘는 데이터로 모두 변경한다.  
    - [참고 : 분산 서버는 왜 홀수개인가?](https://m.blog.naver.com/PostView.nhn?blogId=alice_k106&logNo=220614228476&proxyReferer=https:%2F%2Fwww.google.com%2F)

### AWS의 스토리지 서비스
- 스토리지 서비스가 다양한 이유는 ?
  - 데이터의 형식이 다 다르기 때문이다.

- EBS(Elastic Block Store)란 무엇인가?

    - EC2에 사용되는 하드디스크
    - 여러개의 EBS는 한 개의 ec2에 연결될 수 있다. 다만, 여러개의 ec2는 한 개의 ebs에는 연결할 수 없다.
    - 하나의 하드디스크에 여러개의 컴퓨터를 연결하는 도구를 NAS(Network Attached Storge)라 한다.

- NAS란 무엇인가?
  - 네트워크에 연결된 스토리지 기기이다.
  - 데이터를 한 곳에서 관리하고 네트워크를 통해 여러명이 원격으로 접속이 가능한 환경이다.

- Glacier의 역할은 무엇인가?

    - 콜드 데이터(오랜 시간 사용하지 않는 데이터)를 보관하는 스토리지이다. 비용은 저렴하지만 성능이 좋지 않다. 보통 오래된 로그가 들어간다.

- 스노우볼 서비스는 무엇인가?
  - AWS s3로 데이터 이동을 도와주는 서비스이다.

- AWS Storage Gateway 서비스는 무엇인가?
  - AWS와 회사를 직통으로 연결해 데이터를 옮기는 서비스이다.

### S3 서비스

- 리전 기반 서비스는 무엇이 있는가?
  - S3, VPC, ??

- 백엔드에서 s3를 이용하는 경우는 무엇이 있는가?
  - 로그 파일, artifact(빌드 결과물), 데이터, DB백업 데이터

- S3의 과금 정책은 어떻게 되는가?

    - 저장 용량, 객체 수, 네트워크 트래픽 등으로 과금된다.
    - 같은 리전 내 다른 서비스와 트래픽은 무료 (EC2와 연동 가능)

- EC2에서 S3에 접근하는 법
  - 실제 ec2에서 접근하는 방법은 안전하지 않은 방법. access key가 노출될 수 있기 때문이다.
  - EC2 만들 때 S3에 접근할 수 있는 IAM role을 지정해 s3에 접근할 수 있게 만들어준다.

![ec2만들 때 IAM을 부여하는 사진](/assets/ec2/ec2만들%20때%20IAM을%20부여하는%20사진.png)

- S3의 검색 방식은 무엇인가?
  - S3는 key를 기반으로 객체를 분산 저장하는 분산 저장소이다.
  - key(url)를 통해 객체를 찾는다.
  - 여러 prefix로 분산 처리하면 성능이 좋아진다.

- prefix로 분산처리하는 방식은 무엇인가?

    - 중복되지 않는 prefix를 만들어서 파일을 관리한다. 이래야 분산이 되어 있기 때문에 해시 충돌 가능성이 적어지기 때문이다.

- s3 성능 개선하기 위한 방법 3가지는?
  - 중복되지 않는 여러 prefix로 객체를 나눈다.
  - cloudFront(CDN) 를 사용해서 리소스 접근 속도를 높인다.
  - S3 가속 기능을 이용. 가격은 비쌈.

- S3 여러 클래스
  - 사용자의 상황에 맞게 여러가지 클래스를 선택할 수 있다.

- presignedURL은 무엇인가?

  - 일시적으로 URL에 해당하는 자원에 접근할 수 있는 URL이다.
  - 예를 들어, 넷플릭스 영상 다운로드 링크는 2~3시간 시간 제한이 있다.

- 데이터에 사용자가 많이 접근할 때 서로간 P2P로 다운받게하는 설정을 할 수 있다.

    - url에 ?current를 붙인다?
