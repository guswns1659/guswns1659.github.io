# 금융 도메인 용어 정리

## 통신 방식
금융사는 보안상 public망을 사용하지 않는다. 대신 전용선을 이용한다. 각 금융사의 서버를 관리하는 장소가 있다. KT와 같은 프로바이더가 관리한다고 한다. 클라이언트과 금융사는 해당 전용선을 대여해서 사용한다. 금액은 절반씩 부담한다. 전용선은 평문으로 오고 가기 때문에 VPN을 이용해 패킷을 암호화하는 단계를 추가한다. 네트워크 단에서 암호화를 진행하기 때문에 개발자는 편하다.

통신 방식은 다양한게 선택 가능하다. VPN만 선택할 수 있고 전용선만 선택할 수 있고, 둘 다 선택할 수 있다. 보통 두개 함께 사용하는 걸로 알고 있다.

## 용어 설명

### CI, DI
금융 시스템에서 고객의 고유번호이다. 특정 key를 가지고 고객의 주민번호를 해싱한 값이다. 특정 key는 특정 기관(어디??)에서 제공한다. 회원 가입 시 DB 테이블에 삽입한다. 사용자가 제3자 정보제공동의를 승인하면 CI를 주고 받는 다는 것에 동의하는 것이다.

### OAuth2
제 3기관에게 사용자의 인증을 확인하는 인증 과정이다. 여기서 제 3기관은 신뢰있는 기관이다. 구글, 네이버, 카카오 등과 같은 회사들이다. 사용자가 로그인을 하면 서버는 해당 사용자를 제 3자 기관에 리다이렉트한다. 사용자가 제 3자 기관에서 로그인이 성공하면 accessCode가 서버에 전달된다.
서버는 accessCode를 가지고 사용자에 대한 정보(이름, 아이디 등)를 요청할 수 있는 accessToken과 accessToken을 갱신하는 refreshToken을 제 3기관에 요청할 수 있다.

accessCode, accessToken이 헷갈릴 수 있는데, accessCode는 accessToken을 얻을 때 사용한 도구이다. refreshToken은 accessToken의 짧은 유효기간을 갱신하는데 사용한다. accessToken으로 사용자의 모든 정보를 접근할 수 있기 때문에 제 3기관에서는 그 유효기간을 보통 5분 정도로 짧게 설정한다.

CI와 OAuth2를 함께 연동하는 방법도 있다. 고객의 CI가 있다는 건 해당 고객의 인증이 완료된 상태라는 의미이다. accessCode 대신 CI를 이용해 accessToken을 요청할 수 있다.  

## 전문방식
전문 통신 방식이란 통신에 참여하는 애플리케이션들이 주고 받을 데이터의 포맷을 서로 약속(프로토콜)한 후 약속된 데이터 패킷을 전송하고 수신하는 것을 말한다. 포맷은 plainText, XML, JSON으로 다양하다.

## 배치
정해진 시간에 한번에 데이터를 주고 받는 것을 의미한다. 주로 파일에 데이터를 작성하고 한번만 보낸다. 단점은 실시간으로 이루져야 하는 서비스에는 적용하기 어렵다. 배치를 주고 받는 관계자들은 어떻게 배치를 주고 받을 건지 정한다. 예를 들어, 배치 파일을 만들어 놓을테니 가져가라 하는 방식이 있거나 배치 파일을 만들어서 전달하는 구조도 있다.

## 분리보관
회원이 탈퇴한 후 90일(부정확) 동안 관련 데이터를 가지고 있는 것을 의미한다. 해당 데이터의 민감정보는 제거하고 탈퇴 후에도 사용될 수 있는 데이터만 남긴다. 다른 테이블에 옮겨놓고 시간이 지나면 지운다. 예를 들어, 송금 앱에서 탈퇴하면 내가 가지고 있던 돈을 계좌로 받아야 하는 상황이 있다.

## 대사(reconcile)
데이터의 정합성을 금융사와 맞추는 작업을 의미한다. 데이터를 제공하는 원본을 신뢰해야 한다. 예를 들어, 카드사에서 어제 회원가입한 회원이 10명이라고 하는데 우리 DB에는 8명이라면 10명에 맞춰야 한다.

# 신용카드 도메인 용어

- 승인 : 가맹점의 결제 요청에 따라, 카드사가 카드의 사용가능 여부 및 신용한도 확인
- 한도차감 : 승인이 통과되면 한도에서 결제 금액 차감
- 매입 : 카드사에서 대금지불
- 청구 : 결제일의 카드값

# 오픈뱅킹

오픈뱅킹은 하나의 앱으로 여러 은행의 계좌를 조회하고, 결제∙송금 등을 할 수 있는 금융 서비스입니다. 다시 말해 은행과 핀테크 기업이 공동으로 이용할 수 있는 ‘공동결제 시스템’

은행의 모바일뱅킹 앱이나 핀테크 기업의 금융 서비스 앱은 각 기관에서 자체 개발해 운영해 왔는데요. 오픈뱅킹은 모든 은행을 비롯한 다양한 금융 기관이 공통으로 사용하는 표준화된 송금·결제 망을 활용하는 체제입니다. 즉 오픈뱅킹이 시행되면서 하나의 앱으로 여러 은행 계좌를 관리할 수 있게 되었습니다.
